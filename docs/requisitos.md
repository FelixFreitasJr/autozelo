# Requisitos do Sistema

## 1. Introdução

O AutoZelo é um sistema desenvolvido para auxiliar proprietários de veículos no gerenciamento da manutenção preventiva e corretiva.

O sistema permitirá registrar informações importantes sobre cada veículo, controlar gastos, acompanhar revisões e emitir alertas para futuras manutenções.

---

# 2. Objetivo

Fornecer uma ferramenta simples e organizada para registrar todo o histórico de um veículo, evitando esquecimentos e facilitando o planejamento de futuras manutenções.

---

# 3. Público-alvo

- Proprietários de veículos
- Motoristas de aplicativo
- Pequenas empresas
- Oficinas mecânicas (futuro)

---

# 4. Requisitos Funcionais

## RF001

O sistema deverá permitir cadastrar veículos.

## RF002

O sistema deverá listar todos os veículos cadastrados.

## RF003

O sistema deverá permitir editar informações de um veículo.

## RF004

O sistema deverá permitir excluir veículos.

## RF005

O sistema deverá registrar manutenções.

## RF006

O sistema deverá listar o histórico de manutenções.

## RF007

O sistema deverá registrar a quilometragem atual.

## RF008

O sistema deverá calcular a próxima manutenção.

## RF009

O sistema deverá registrar despesas.

## RF010

O sistema deverá emitir alertas de manutenção.

---

# 5. Requisitos Não Funcionais

## RNF001

O sistema deverá ser desenvolvido em Java.

## RNF002

O código deverá seguir princípios de Orientação a Objetos.

## RNF003

O projeto deverá utilizar Git para versionamento.

## RNF004

O sistema deverá possuir documentação técnica.

## RNF005

O sistema deverá permitir expansão para banco de dados futuramente.

---

# 6. Restrições

Na primeira versão o sistema funcionará apenas em modo console.

Não haverá banco de dados.

Os dados permanecerão apenas durante a execução do programa.
